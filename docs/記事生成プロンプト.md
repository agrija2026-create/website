# 記事生成プロンプト（HTML肥大化対策版）

記事化時は、以下の条件を満たす HTML 記事だけを生成すること。SEO とクロール効率を守るため、**本文 HTML を軽く保つこと**を最優先にする。

## 目的

- 農業政策・制度・補助金・予算の情報を、一次情報ベースでわかりやすく整理する
- サイト側で共通表示する要素と、本文側で書く内容を分離する
- Google が読み取る HTML を肥大化させない

## 出力形式

- 完成物は **HTMLファイル 1つ**
- `<article>...</article>` の中に本文を書く
- `<h1>` と先頭 `<header>` は原則書かない
- 冒頭に 1 段落で記事の要点をまとめる
- 見出しは `<h2>` と必要に応じて `<h3>` を使う
- 末尾に出典をまとめる

## 絶対に守る制約

1. **`data:image` や Base64 画像を、そのまま公開しない**
2. **画像バイナリを HTML に埋め込んだままにしない**
3. SVG の data URI、CSS の `url(data:...)` も使わない
4. 巨大なインライン CSS や JavaScript を入れない
5. PDF をそのまま巨大なスクリーンショットとして貼り付けない
6. 図版は使ってよいが、**軽量な外部画像ファイルとして扱う**

## 図版の扱い

- **引用元資料の図版掲載は可**。むしろ読者理解に必要な図版は入れてよい
- 可能なら、図版は最初から外部画像ファイルとして扱う
- もし作業途中で `data:image` を使っても、公開前に外部画像へ変換して軽量化する
- 図版は必要最小限に絞り、同じ内容のキャプチャを何枚も並べない
- 可能なら `WebP` または圧縮済み `PNG/JPEG` を使う
- 横幅が極端に大きい画像は縮小する
- 1つの図で済む内容を複数画像に分割しない
- 図版だけに依存せず、本文でも要点が分かるように補足を書く
- 図版には `figcaption` で何を示す図かを必ず書く

## 本文量と構造の制約

- 1記事で扱う論点は絞る
- 1つの表が極端に長くなる場合は、表を分割する
- 制度比較は「対象」「支援内容」「補助率」「注意点」など、比較軸を固定して表にする
- 表の基本ルールは **「2列表の要点整理」か「3〜4列の比較表」** に寄せる
- **5列以上の表は原則作らない**。必要なら 2つの表に分割する
- 冒頭の要点表は 2列表にし、左列は **短い見出し語（例: 誰が / 何を / いつまでに / いくら）** にそろえる
- 左列の見出しは、可能なら **4〜12文字程度** に収める。長い見出しは見出し本文側に逃がす
- 比較表は 1セルに情報を詰め込みすぎない。1セルが 3行を大きく超えるなら表の分割を優先する
- 同じ説明の繰り返しを避ける
- 不要な装飾HTMLは使わない

## 文体・編集方針

- 一次情報に基づいて書く
- 断定しすぎず、要件や例外は明示する
- 読者が判断しやすい順で整理する
- 主観的な煽り表現は避ける
- 「詳細は一次情報をご確認ください」のように、確認先を明示する

## 記事に含めるべき要素

- この記事の対象者
- 制度や施策の概要
- 対象になる条件
- 補助率、上限額、期限など重要数値
- 注意点、誤解しやすい点
- 必要な図版とその説明
- 出典名と URL

## 望ましい冒頭の型

```html
<article>
  <p class="lead">この記事では、◯◯の対象者、支援内容、補助率、注意点を整理します。詳細な要件や最新情報は一次情報をご確認ください。</p>

  <h2>まず押さえたい要点</h2>
  <ul>
    <li>...</li>
    <li>...</li>
  </ul>

  <h2>対象者</h2>
  <p>...</p>

  <h2>支援内容</h2>
  <p>...</p>

  <figure>
    <img src="図版ファイルのURLまたはパス" alt="図版の内容を簡潔に説明">
    <figcaption>図版が何を示しているかを簡潔に説明する。</figcaption>
  </figure>

  <h2>注意点</h2>
  <p>...</p>

  <p class="source">【出典】資料名、URL</p>
</article>
```

## AI への指示文テンプレート

以下をそのまま記事生成時のベース指示として使える。

```text
農業政策・制度の解説記事を HTML で作成してください。

要件:
- 出力は HTML のみ
- 本文は <article>...</article> 内に書く
- <h1> と先頭 <header> は使わない
- 冒頭に要点をまとめた短い導入を書く
- 見出しは h2 / h3 を使う
- 読者が判断しやすい順で整理する
- 一次情報ベースで、対象者、支援内容、補助率、上限額、注意点、確認先を明記する
- 読者理解に必要な図版は入れてよい
- 図版には alt と figcaption を付ける
- 末尾に出典名とURLを入れる

禁止事項:
- data:image や base64 画像を、そのまま公開しない
- 画像バイナリを HTML に埋め込んだままにしない
- SVG の data URI を使わない
- 巨大な inline style や script を入れない
- 高解像度すぎる画像や重い画像をそのまま使わない

図表方針:
- 必要な図版は使ってよい
- ただし図版は公開時に軽量な外部画像として扱う
- 表は 2列の要点表、または 3〜4列の比較表に寄せる
- 5列以上の表は作らず、必要なら複数表に分ける
- 冒頭の要点表の左列は短い語でそろえる
- 長い表は分割する
- 画像だけに頼らず、本文でも意味が通る記事にする

文体:
- 断定しすぎず、要件や例外を丁寧に書く
- 誇張表現を避ける
- 最後に「詳細は一次情報をご確認ください」と確認先を示す
```
