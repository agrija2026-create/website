## 1. 本番・ブランド

| 項目 | 内容 |
|------|------|
| サイトURL（正規の想定・構造化データ等） | https://agri-ja.net/ |
| 実配信ドメイン | 切替まで旧ドメインのまま運用してもよい |
| X（旧Twitter） | https://x.com/agri_ja |
| サイト名 | 農業情報メディア（メタ説明は `app/layout.tsx`） |

環境変数 `NEXT_PUBLIC_SITE_URL` でオリジンを上書き可能（未設定時は **https://agri-ja.net**）。**canonical・OG・JSON-LD** はここを参照。ドメイン切替前でも、構造化データは新アドレスでよい、という方針。

**sitemap.xml / robots.txt** はドメイン切替まで **リポジトリから外しておく**（配信しない）。切替後に `app/sitemap.ts` / `app/robots.ts` を足して Search Console に登録する想定。

## 3. 読者向けタグ（実装済み）

次の3つだけを「読者タグ」として `tags` に付与（**必須・1〜3個**）。`lib/tags.ts` の `AUDIENCE_TAGS` で固定。

- `生産者向け` / `小売向け` / `流通向け`

- **検証**: `lib/articles.ts` がローカル・microCMS いずれの記事でもパース時にチェック。違反時はビルド・開発サーバー起動時に例外。
- **UI**: サイドバー「読者別」＋「タグ一覧」から読者3つは除外。記事カード・記事ページで「対象読者」と「タグ」を分離表示。
- **URL**: 読者タグの一覧ページは日本語パスではなく ASCII にマッピング（`/tags/reader-producers` 等）。表示ラベルは日本語のまま。

---

## 4. 記事運用（量産時）

- **優先**: 読者の判断に直結するテーマ、公式の新規・改定、内部リンクでつながるテーマ。
- **品質下限**: 出典URL・資料名・確認日、読者タグ、用語の初出定義、該非の線引き。出典なし数値・断定のみは禁止。
- **型**: 冒頭で結論と対象者。改正時は同一URL更新＋最終確認日。
- **ペース**: 監修可能な週次上限を決める（日次連発より安定した週ペース）。
- **AI**: 監修必須。検証時はテーマ・型を揃えプロセスだけ変える。

---

## 5. 内部リンク

**通常（記事がある状態）**

- 同カテゴリの既存記事へ **1本以上**
- 読者タグが1つ以上重なる既存記事へ **1本以上**（上と同一可）
- カテゴリ一覧（ハブ）へ **1本**
- アンカーは自然な語。数合わせリンクはしない。

**サイト全体の最初の1本**

- 記事相互は **0本可**。カテゴリハブやトップへの導線は任意で1本。

**そのカテゴリの最初の1本**

- 「同カテゴリ1本」は **免除**。別カテゴリの関連1本＋読者が重なる記事1本（同一可）を目安。

**2本目以降**

- カテゴリ2本目で **1本目へリンク**、可能なら1本目に2本目を追記。

## 6. SEO インフラ（実装状況）

| 項目 | 内容 |
|------|------|
| metadataBase | `app/layout.tsx`（本番オリジン） |
| JSON-LD Organization | 全ページ（`OrganizationJsonLd`） |
| JSON-LD Article + BreadcrumbList | 記事ページ（`ArticleStructuredData`） |
| canonical / OG | 記事 `generateMetadata`（正規URLは `agri-ja.net` 基準） |
| sitemap.xml | **未配置**（ドメイン切替後に追加予定） |
| robots.txt | **未配置**（同上） |

**未対応（検討リスト）**: Article の `image`、`dateModified` を frontmatter で持つ、Organization の `logo`、サイト内検索用 WebSite + SearchAction。

## 7. 構造化データで出している型

- **Organization**: name, url, sameAs（X）, `@id`
- **Article**: headline, description, datePublished / dateModified（現状は公開日ベース）, author/publisher → Organization 参照, mainEntityOfPage
- **BreadcrumbList**: トップ → カテゴリ → 記事
