## 2. カテゴリ（8本・英語 slug）

| slug | 表示名 |
|------|--------|
| policy | 政策・制度 |
| budget | 予算・財政 |
| market | 市場・価格・需給 |
| logistics | 流通・物流 |
| production | 生産・作物（野菜・園芸） |
| farmland | 農地・担い手・経営 |
| technology | 技術・DX・スマート農業 |
| food-safety | 表示・規格・食品安全 |

マスタ: `lib/categories.ts`。URL: `/categories/{slug}`。

### カテゴリの付け方（判定ルール）

記事の frontmatter `category` は **1記事1カテゴリ**。フォルダ分けではなく各 `.md` に slug（英語）で指定する。

**優先順位（上から判定）**

1. **補助金・資金が主題** → `budget`  
   交付金、補助金、資金、融資、補正予算、事業の補助率・上限額が記事の中心なら、テーマが農地・有機でも `budget`。
2. **輸出・貿易・需給** → `market` または `policy`  
   - `market`: 輸出促進事業、海外需要、関税、需給・シェア、価格・出荷見通し  
   - `policy`: 国と都道府県/JA の連携体制など制度の枠組み
3. **流通・サプライチェーン** → `logistics`  
   商慣習見直し、流通再編、食品ロス（供給 chain 視点）
4. **規格・認証・食品安全** → `food-safety`  
   GAP、JAS、HACCP、輸出先国規制対応
5. **生産・作物** → `production`  
   病害虫予報、種子供給、作物・生育（補助金が主題でないもの）
6. **農地・担い手・経営** → `farmland`  
   農地バンク、就農制度の仕組み、経営・担い手・女性活躍
7. **技術・DX** → `technology`  
   ドローン散布、スマート農業
8. **上記に当てはまらない横断政策** → `policy`  
   基本計画、法律の全体像、食料危機対策法、環境・食産業の横断政策

**stage 時**: `article:stage` の第4引数で category を必ず指定する（未指定時は `policy` になるため偏りやすい）。一括見直しは `scripts/recategorize-articles.py` を参照。

## 3. 読者向けタグ（実装済み）

次の3つだけを「読者タグ」として `tags` に付与（**必須・1〜3個**）。`lib/tags.ts` の `AUDIENCE_TAGS` で固定。

- `生産者向け` / `小売向け` / `流通向け`

- **検証**: `lib/articles.ts` がローカル・microCMS いずれの記事でもパース時にチェック。違反時はビルド・開発サーバー起動時に例外。
- **UI**: サイドバー「読者別」＋「タグ一覧」から読者3つは除外。記事カード・記事ページで「対象読者」と「タグ」を分離表示。
- **URL**: 読者タグの一覧ページは日本語パスではなく ASCII にマッピング（`/tags/reader-producers` 等）。表示ラベルは日本語のまま。

## 4. 記事運用（量産時）

- **優先**: 読者の判断に直結するテーマ、公式の新規・改定、内部リンクでつながるテーマ。
- **品質下限**: 出典URL・資料名・確認日、読者タグ、用語の初出定義、該非の線引き。出典なし数値・断定のみは禁止。
- **型**: 冒頭で結論と対象者。改正時は同一URL更新＋最終確認日。
- **ペース**: 監修可能な週次上限を決める（日次連発より安定した週ペース）。
- **AI**: 監修必須。検証時はテーマ・型を揃えプロセスだけ変える。

## 5. 内部リンク

**通常（記事がある状態）**

- 同カテゴリの既存記事へ **1本以上**
- 読者タグが1つ以上重なる既存記事へ **1本以上**（上と同一可）
- カテゴリ一覧（ハブ）へ **1本**
- アンカーは自然な語。数合わせリンクはしない。

**サイト全体の最初の1本**

- 記事相互は **0本可**。カテゴリハブやトップへの導線は任意で1本。

**そのカテゴリの最初の1本**

- 「同カテゴリ1本」は **免除**。別カテゴリの関連1本＋読者が重なる記事1本（同一可）を目安。

**2本目以降**

- カテゴリ2本目で **1本目へリンク**、可能なら1本目に2本目を追記。

## 6. SEO インフラ（実装状況）

| 項目 | 内容 |
|------|------|
| metadataBase | `app/layout.tsx`（本番オリジン） |
| JSON-LD Organization | 全ページ（`OrganizationJsonLd`） |
| JSON-LD Article + BreadcrumbList | 記事ページ（`ArticleStructuredData`） |
| canonical / OG | 記事 `generateMetadata`（正規URLは `agri-ja.net` 基準） |
| sitemap.xml | **配置済**（`public/sitemap.xml`。`npm run sitemap:generate` で `content/articles/*.md` から生成。build / publish 時に更新） |
| robots.txt | **配置済**（`app/robots.ts` → `/robots.txt`。全クロール許可 + sitemap 参照） |

**未対応（検討リスト）**: Article の `image` を記事別に指定、Organization の `logo`、サイト内検索用 WebSite + SearchAction。

## 7. 構造化データで出している型

- **Organization**: name, url, sameAs（X）, `@id`
- **Article**: headline, description, datePublished / dateModified（`updatedAt` 未指定時は公開日）, author/publisher → Organization 参照, mainEntityOfPage, articleSection, inLanguage, keywords / about（テーマタグ）, isBasedOn（出典 URL・HTML の source ブロックから自動抽出）
- **BreadcrumbList**: トップ → カテゴリ → 記事
